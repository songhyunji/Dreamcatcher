일반적인 RPG에서 사용하는 인벤토리는 저희가 구현하려는 인벤토리에 비해 기능이 과하다고 판단하였습니다.
(ex. 아이템 사용, 아이템 버리기, 아이템 개수 ... etc)
아이템의 설명을 보여주는 용도로만 사용할 것이라 예상하여 그에 맞춰 기능을 바꿨는데요,
게임 진행상 모든 아이템은 순서대로, 반드시 취득하게 되어있기 때문에
모든 아이템은 미리 값을 받아놓되, SetActive 값을 false로 지정하게 할 생각입니다.
나중에 특정 부분에서 해당 슬롯의 SetActivce값을 true로 설정하면 해당 아이템이 해금되듯이 보여지는 것으로요.
하지만 제가 데이터베이스에 대해 지식이 전무하고(이번학기 때 배우는 전공이라서요 ㅠㅠ)
미리 만든 값을 저장하는 방법은 적용했는데, 이 값을 수정하려면 친구 말로는 DBSM이 필요하다.. 그러더라고요. (안배움..)
그래서 헤매다가 그냥 이 부분은 냅뒀습니다..ㅠㅠ 내일 다른 개발파트분들 조언 부탁드릴게요.,, 흑흑
반복되는 코드의 경우 JetBrains 플러그인으로 복사를 하신 것 같은데 일단은 Scene1에만 인벤토리가 나타나게 해두었습니다.
